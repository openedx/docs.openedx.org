Library Roles
#############

.. tags:: educator, reference

following roles can be :ref:`assigned to each user on a library team <Add users to Libraries>`.

.. _Library User:

Library User
************

Library Users can:

* View published content in the library

* Search, filter and sort library content

* Reuse published content from the library in their courses

They cannot create, edit or delete content, manage tags or collections, or change the library team.

.. _Library Contributor:

Library Contributor
*******************

Library Contributors can:

* View library content

* Create new content and edit content in draft

* Import content from courses into the library

* Create collections and add or remove content from collections

* Add and remove tags from content and from collections

* Reuse content in courses

* View who else is on the library team

They cannot publish content, delete libraries or manage roles and team
membership. Publishing remains the responsibility of Library Authors and Library
Admins.

.. _Library Author:

Library Author
**************

Library Authors can:

* Do everything a **Library Contributor** can do

* Publish content in the library so it becomes available for reuse

Authors are responsible for managing the lifecycle of content in the library,
from draft to published, but they cannot delete entire libraries or manage the
library team.

.. _Library Admin:

Library Admin
=============

Library Admins have full control over the library. They can:

* Do everything a **Library Author** can do

* Add and remove users from the library team

* Change roles for existing team members

* Delete entire libraries

* Toggle a library between editable and read only states

Users who have global admin access on the platform also have Library
Admin permissions on all libraries, but will not appear in the library team
list unless they are manually added as a team member.
